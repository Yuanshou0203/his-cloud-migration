## 二、数据库设计与测试数据

### 2.1 表结构（4张表）

| 表 | 说明 | 敏感字段 |
|----|------|----------|
| patients | 患者信息 | 身份证号、手机号、住址 |
| drugs | 药品字典 | 价格、库存 |
| prescriptions | 处方 | 诊断、总金额 |
| prescription_items | 处方明细 | 用法用量 |

### 2.2 创建库表

```bash
mysql -u root -proot
```

```sql
CREATE DATABASE his_hospital CHARSET utf8;
USE his_hospital;

CREATE TABLE patients(
  id INT PRIMARY KEY AUTO_INCREMENT,
  medical_record_no VARCHAR(20) NOT NULL UNIQUE,
  name VARCHAR(50) NOT NULL,
  gender ENUM('男','女') NOT NULL,
  birth_date DATE NOT NULL,
  id_card VARCHAR(18) NOT NULL,
  phone VARCHAR(15) NOT NULL,
  address VARCHAR(200)
);

CREATE TABLE drugs(
  id INT PRIMARY KEY AUTO_INCREMENT,
  drug_code VARCHAR(20) NOT NULL UNIQUE,
  drug_name VARCHAR(100) NOT NULL,
  specification VARCHAR(50),
  manufacturer VARCHAR(100),
  price DECIMAL(10,2) NOT NULL,
  stock INT DEFAULT 0
);

CREATE TABLE prescriptions(
  id INT PRIMARY KEY AUTO_INCREMENT,
  patient_id INT NOT NULL,
  doctor_name VARCHAR(30) NOT NULL,
  diagnosis TEXT,
  total_amount DECIMAL(10,2),
  status ENUM('已发药','待发药','已退费') DEFAULT '待发药'
);

CREATE TABLE prescription_items(
  id INT PRIMARY KEY AUTO_INCREMENT,
  prescription_id INT NOT NULL,
  drug_id INT NOT NULL,
  quantity INT NOT NULL DEFAULT 1,
  dosage VARCHAR(50)
);
```

### 2.3 插入测试数据

- 10 名患者（含真实格式身份证号、手机号）
- 10 种药品（涵盖常见处方药：阿莫西林、二甲双胍、硝苯地平、奥美拉唑等）
- 15 张处方（含诊断信息：糖尿病、高血压、高脂血症、冠心病等）
- 21 条处方明细（药品-处方关联）

## 三、分级权限管控

```sql
-- DBA 账号（全部权限）
GRANT ALL PRIVILEGES ON his_hospital.* TO 'his_dba'@'%' IDENTIFIED BY 'HisDba@2024';

-- 只读账号（报表/查询）
GRANT SELECT ON his_hospital.* TO 'his_readonly'@'%' IDENTIFIED BY 'HisRead@2024';

-- 应用账号（CRUD，不能改表结构）
GRANT SELECT, INSERT, UPDATE, DELETE ON his_hospital.* TO 'his_app'@'%' IDENTIFIED BY 'HisApp@2024';

-- 审计账号（仅患者和处方表）
GRANT SELECT ON his_hospital.patients TO 'his_audit'@'%' IDENTIFIED BY 'HisAudit@2024';
GRANT SELECT ON his_hospital.prescriptions TO 'his_audit'@'%';
```

**安全效果**：敏感数据分级访问，越权查询自动拒绝。

## 四、MySQL 数据迁移

### 4.1 全量迁移

```bash
# 1. 源端导出（一致性快照 + 记录 binlog 位置）
mysqldump -u root -proot --single-transaction --master-data=2 his_hospital > his_backup.sql

# 2. 安全传输
scp his_backup.sql root@192.168.133.40:/tmp/

# 3. 目标端恢复
mysql -u root -proot -e "CREATE DATABASE his_hospital CHARSET utf8;"
mysql -u root -proot his_hospital < /tmp/his_backup.sql

# 4. 验证
mysql -u root -proot his_hospital -e "SELECT COUNT(*) FROM patients;"
# 源: 10  目标: 10  ✅ 一致
```

| 参数 | 作用 |
|------|------|
| --single-transaction | InnoDB 一致性快照，不锁表 |
| --master-data=2 | 记录 binlog 位置，注释形式写入备份文件 |

### 4.2 增量同步（binlog）

```bash
# 配置源端 binlog
cat >> /etc/my.cnf << EOF
log-bin=mysql-bin
binlog_format=ROW
server-id=1
expire_logs_days=7
EOF
systemctl restart mariadb

# 获取当前 binlog 位置
mysql -u root -proot -e "SHOW MASTER STATUS;"

# 导出增量
mysqlbinlog --start-position=<pos> /var/lib/mysql/mysql-bin.000001 > incremental.sql

# 应用到目标
mysql -u root -proot his_hospital < incremental.sql
```

## 五、Nginx WAF 边界防护

### 5.1 限流配置

```nginx
limit_req_zone $binary_remote_addr zone=his_api:10m rate=30r/s;
limit_conn_zone $binary_remote_addr zone=his_conn:10m;
```

每 IP 每秒最多 30 请求，突发缓冲 50，最大并发连接 50。

### 5.2 SQL 注入拦截

```nginx
if ($query_string ~* "union[\s+]+select|select[\s+]+from|
    insert[\s+]+into|drop[\s+]+table|delete[\s+]+from|
    update[\s+]+.*set|--|/\*") {
    return 403;
}
```

**拦截规则覆盖**：UNION 注入、堆叠查询、注释绕过、系统表读取。

### 5.3 XSS 拦截

```nginx
if ($request_uri ~* "<script[^>]*>|javascript:|onerror=|
    onload=|alert\(|eval\(|document\.cookie") {
    return 403;
}
```

### 5.4 文件上传控制

```nginx
if ($request_uri ~* "\.(php[0-9]?|jsp|asp|aspx|cgi|pl|py)$") {
    return 403;
}
client_max_body_size 10m;
```

### 5.5 目录遍历防护

```nginx
if ($request_uri ~* "\.\./|\.\.\\") {
    return 403;
}
```

### 5.6 安全响应头

```nginx
add_header X-Frame-Options "DENY" always;           # 防点击劫持
add_header X-Content-Type-Options "nosniff" always; # 防 MIME 嗅探
add_header X-XSS-Protection "1; mode=block" always; # 浏览器 XSS 过滤器
```

## 七、容灾备份

```bash
# 定时全量备份（crontab 每日 2:00）
0 2 * * * mysqldump -u root -proot --single-transaction his_hospital > /backup/his_$(date +\%Y\%m\%d).sql

# binlog 保留 7 天（/etc/my.cnf）
expire_logs_days=7
```

| 指标 | 目标 |
|------|------|
| RPO | ≤ 1 小时（binlog 实时） |
| RTO | ≤ 30 分钟（全量恢复） |
