# HIS 业务系统云上迁移 - 实施文档

## 架构

```
本地数据中心 (.20)                    阿里云 (.40)
┌────────────────────┐              ┌──────────────────────┐
│ MySQL (源库)       │  全量+增量   │ MySQL (目标库)        │
│ his_hospital       │─────────────►│ his_hospital          │
│ patients: 10       │  mysqldump   │ patients: 10          │
│ drugs: 10          │  + binlog    │ drugs: 10             │
│ prescriptions: 15  │              │ prescriptions: 15     │
│                    │              │                       │
│ Apache (源站)      │              │ Nginx WAF (入口)      │
└────────────────────┘              │ upstream → Docker:8080│
                                    └──────────────────────┘
```

## 实施步骤

### 1. MySQL 数据迁移
1. 创建 HIS 数据库: patients(患者) / drugs(药品) / prescriptions(处方) / prescription_items
2. 插入测试数据: 10 患者 + 10 药品 + 15 处方 + 21 处方明细
3. 全量导出:
   ```bash
   mysqldump -u root -proot --single-transaction --master-data=2 his_hospital > his_backup.sql
   ```
4. 传输到目标: scp his_backup.sql root@docker-node2:/tmp/
5. 目标恢复: mysql -u root -proot his_hospital < his_backup.sql
6. 验证: 源端和目标端 COUNT(*) 一致 (10 patients)

### 2. 数据库分级权限
| 账号 | 权限 | 用途 |
|------|------|------|
| his_dba | ALL PRIVILEGES | 数据库管理员 |
| his_readonly | SELECT | 报表查询 |
| his_app | SELECT,INSERT,UPDATE,DELETE | 应用程序 |
| his_audit | SELECT(patients,prescriptions) | 合规审计 |

### 3. Nginx WAF 部署

限流: limit_req_zone rate=30r/s, burst=50

SQL注入拦截规则:
```
union select | select from | insert into | drop table |
delete from | update set | -- | /*
```

XSS拦截规则:
```
<script> | javascript: | onerror= | alert( | eval( | document.cookie
```

文件上传控制:
```
拦截 .php .jsp .asp .aspx .cgi .pl .py 等可执行后缀
```

目录遍历防护:
```
拦截 ../ 路径穿越
```

安全响应头:
```
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
```

### 4. 安全复测

| 测试 | 请求 | 预期 | 结果 |
|------|------|------|------|
| SQL注入 | ?id=1 union select 1 | 403 | ✅ 403 |
| XSS | ?q=<script>alert(1)</script> | 403 | ✅ 403 |
| 正常请求 | / | 200 | ✅ |

### 5. 容灾备份
- binlog 配置: binlog_format=ROW, expire_logs_days=7
- 全量备份: mysqldump --single-transaction 每日
- 增量备份: binlog 实时同步
```
